Before making any modifications:

1. Read and explain what the existing code does in plain English
2. Identify any logic that is unclear or potentially fragile
3. Flag anything that looks like a bug or unintended behaviour
4. Summarise what you understand the intent to be

Only proceed with changes after this explanation is confirmed as correct.
